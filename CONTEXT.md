# Project Context: DCAA-Compliant Timekeeping Application

## 1. Project Overview
This project is a modern, fault-tolerant, and highly intuitive Next.js application designed specifically for Government Contractors (GovCon). The goal is to displace legacy systems like Deltek, Unanet, and HourTimesheet by providing a vastly superior User Experience (UX) while strictly adhering to Defense Contract Audit Agency (DCAA) compliance standards.

## 2. Technology Stack
* **Framework:** Next.js (App Router, React Server Components, Server Actions).
* **UI Framework:** Mantine v9 (Strictly enforced) and Mantine React Table v2. Do not mix with other UI component libraries (like shadcn/ui or MUI) to ensure a cohesive theme and dark/light mode parity.
* **Database:** PostgreSQL (Strict relational schema).
* **ORM:** Drizzle ORM (for type-safe, edge-compatible SQL queries).
* **Authentication:** Auth.js (Email/Password for MVP, utilizing short-lived JWTs with role-based payloads, and secure HTTP-only refresh tokens).
* **Offline/Fault Tolerance:** Client-side caching (IndexedDB/Dexie.js or WatermelonDB) to allow offline time logging, syncing automatically once the network connection returns.

## 3. Core Business Rules (DCAA Compliance - NON-NEGOTIABLE)
When building features, prioritize data integrity and auditability. These rules are absolute:
* **Append-Only Architecture:** Once a timesheet entry is saved, it can NEVER be directly `UPDATE`d or `DELETE`d. Any corrections or changes require inserting a *new* row with an incremented `revision_number`, a `change_reason_code`, and the `updated_by` user ID, while preserving the historical record.
* **Daily Time Entry:** Employees must log time daily. The system must record exactly *when* time was entered for audit purposes.
* **Total Time Accounting:** All hours worked must be recorded, including unpaid/uncompensated overtime for FLSA-exempt employees.
* **Granular RBAC:** Employees can ONLY view, access, and log time against Contracts and CLINs (Contract Line Item Numbers) they have been explicitly assigned to.
* **Approval Workflows:** Digital certification is required for employees (submitting) and supervisors (reviewing/approving/rejecting).

## 4. Domain / Data Hierarchy
The system maps complex GovCon structures using strict relational database constraints:
`Contracts` -> `CLINs` (Line Items) -> `SLINs` (Sub-Line Items) -> `Labor Categories` (Roles/Rates) -> `Employees` -> `Timesheet Entries`
